Explanation of Generic Sample Files

In the archetypes/sample projects, there are several files that are included to help get right into testing. Here is an explanation of each of these files. 

.
├── config
│   ├── framework
│   │   ├── StoryContext.java
│   │   └── StoryContextProvider.java
│   └── selenium
│       ├── ConfigurableDriverProvider.java
│       └── DriverProviderProducer.java
├── page
└── step

Page and Step packages - src/test/java/<project>/<page or step>

The files in these packages are the basics of the sample tests. They are there only as a template, and can be removed. See Create Tests for information on how to begin creating tests.

Alternatively, see Page Objects and Step Objects for further information on the objects that live in the page and step packages.

Beans.xml - src/test/resources/META-INF/beans.xml

This file is typically completely blank if using one of the archetypes. It is required to tell the container to scan for weld dependencies.

Some configurations can be set to create more default projects. See Selenium above for an example of this

Create Tests

Due to the the nature of this framework, there will be several different ways to create tests. Creating tests for Cucumber, JBehave, and JUnit will be covered in this section.

Define Test

All tests have a set of instructions that contain the steps necessary to execute the tests. These instructions are different for each framework:

  • JBehave
  • Cucumber
  • JUnit

JBehave calls these sets of instructions story files. They are Given/When/Then statements that are located inside of a .story file. These files are located at src/test/resources/stories/. A example .story file looks like this: 

Search Bing for information

Narrative:
In order to know more about Partnet
As a tester
I want to search for their website 

Meta:
@browser firefox

Scenario: Search bing for Partnet
Given I am on Bing's home page
When I search for Partnet
Then I will see Partnet's home page in the list of results
Lets go though this line by line. The first line is the title of the test.

The Narrative gives a brief overview to the story to describe some functionality.

The Meta tag is a JBehave feature, to allow stories to be tagged for various reasons, including filtering stories. A feature that the SeAuto JBehave sample project provides is launching a specific browser per story. The following @browser meta tags are available, and are not case sensitive:

  • Firefox
  • IE
  • PhantomJs
  • HTMLUnit

In the SeAuto JBehave sample, browsers can't be specified per scenario. One browser is created for the entire story at the story level.

Scenario defines a section of the test

Given/When/Then are the steps and describe what to test. The match directly with annotated methods.

Specifics about the story syntax can be found on JBehave's website.

Cucumber calls these sets of instructions features. They are Given/When/Then statements that are located inside of a .feature file. They are located at src/test/resources/<package-name>/. It is packaged with a direct relation to where the CucumberRunner is located, with the exception of being in the src/test/resources folder instead of src/test/java. 

Feature: Search Bing for information

  @firefox
  Scenario: Search bing for Partnet
    Given I am on Bing's home page
    When I search for Partnet
    Then I will see Partnet's home page in the list of results
Going though this line by line, The first line is the title of the test and needs to start with Feature

The @firefoxannotation specifies what browser to use for that specific test. This is defined for each scenario. If a scenario doesn't have this tag, no browser will be launched for the scenario.

Available annotations:

  • @firefox
  • @ie
  • @phantomjs
  • @htmlunit

Scenario defines a portion of the test. In Cucumber, they are isolated, and do not chain from one scenario to another.

Given/When/Then are the steps and describe what to test. The match directly with annotated methods.

Specifics on the cucumber syntax, called Gherkin, can be found on their wiki page.

In JUnit, these set of instructions are regular java JUnit tests, calling step methods. A sample JUnit test looks similar to this:

@RunWith(SeAuto.class)
public class BingTest
{

  @Inject
  private SearchSteps searchSteps;
  
  @Test
  @Firefox
  public void searchBingWithFirefox()
  {
    searchSteps.givenIAmOnBingsHomePage();
    searchSteps.whenIsearchForPhrase("Partnet");
    searchSteps.thenIWillSeePartnetInResults();
  }
}

The JUnit test will be extremely familiar for anyone who has dealt with them before.

The tests need to be run with the SeAuto.class, steps need to be injected into the test, and a test method needs to be annotated with what browser one would like to use to test with.

Available browser annotations:

  • @Firefox
  • @IE
  • @PhantomJs
  • @HTMLUnit

Other then that, they are just like regular unit tests. Each test is isolated, and should be written to be executed in any order.

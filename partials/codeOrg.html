Archetype Code Organization

Archetypes have some basic structures to them. It is a suggestion to layout the basis of your automated tests in this way

In the src/test/java package

  • config - responsible for the configuration and setup of the framework
  • page - responsible for the page and site objects
  • step - responsible for the step objects

In src/test/resources

  • META-INF - beans.xml to allow Weld to work
  • stories - this is where the .story files are located. .story files is where the written individual steps are located
  • config.properties - the properties for the running tests.
  • log4j.properties - the logger properties for the running tests.

In resources/

  • selenium - the binaries for the various drivers
